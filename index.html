Verschiedene Strategien und Arten von Angriffen

  • Malware
  • Phishing
  • SQL Einschleusung (SQL Injecion)
  • Cross-Site Scripting (XSS)
  • Session Hijacking und Man-in-the-Middle Attacks
  • Social engineering
  • Denial of Service
  • Malware: Antivirus Warnung im Browser, bösartiger E-Mail Anhang installiert Viren oder Ransomware am Computer
  • Phishing: Versuch vertrauliche Daten abzugreifen, in dem man zum Beispiel den Benutzer auf eine fingierte Website weiter leitet.
  • SQL Injection: Über eine SQL (Structured Query Language) Verwundbarkeit wird bösartiger Code in die Datenbank eingeschleust. Das kann zB über ein Suchformular erfolgen, in das der bösartige Code eingegeben und der SQL Server gezwungen wird, vertrauliche Daten auszugeben
  • XSS: Malware läuft nur im Browser des Besuchers und greift den Besucher direkt an, zB durch die Eingabe von bösartigem Code in einen Kommentar
  • Session Hijacking: Wenn zwei Computer über das Internet miteinander kommunizieren, wird eine Session eröffnet, die eine einzigartige ID bekommt. Diese ID sollte vertraulich gehalten werden, da über diese ID sicher gestellt wird, dass man mit dem "richtigen" Adressat kommuniziert. In einer Man-in-the-Middle Attacke wird diese Session ID sozusagen entführt bzw gestohlen. Der Angreifer gibt sich nun als der entfernte Server aus und versucht so zu vertraulichen Informationen zu kommen.
  • Social engineering: durch zwischenmenschliche Beeinflussung versucht man bestimmte Verhaltensweisen hervorzurufen, um zum Beispiel zur Preisgabe von vertraulichen Informationen oder zur Freigabe von Finanzmitteln zu bewegen. Social Engineering wird auch eingesetzt, um in fremde Computersysteme einzudringen.
  • Denial of Service: Server und IT-Systeme werden mit einer so großen Anzahl von Anfragen beschossen bis sie ihren Dienst einstellen und Websites nicht mehr aurufbar sind.

Enormer Anstieg von DDOS Angriffen

  • Mirai Botnet und das Internet der Dinge

Unter Mirai versteht man eine Linux-Schadsoftware, mit derman Bot-Netze aufbauen kann (= Gruppe automatisierter Schadprogramme). Mit diesen lassen sich gezielte Attacken durch absichtliche Überlastungen von Netzen ausführen.

Mirai nutzt die Tatsache aus, dass immer mehr Alltagsgegenstände wie Router, CCTV Überwachungssysteme, Digitale Video Recorder, Fernsehrer, DVD Player usw. mit dem Internet verbunden sind (Internet der Dinge).

Die Software scannt das Netz nach Sicherheitslücken vor allem bei Internet der Dinge Geräten und versucht dann, Schadcode auf diese aufzuspielen.

Am 21. Oktober 2016 wurde mit Mirai ein großangelegten DDoS-Angriffen auf die Firma Dyn durchgeführt. Viele große Webseiten, welche das DNS Service von Dyn nutzten, waren dadurch stundenlang nicht erreichbar.

Die einzigen großen Websites, die nicht davon betroffen waren, waren jene der Pornoindustrie – weil deren Webserver auf einen alternativen Anbieter umschalteten als Dyn nicht mehr erreichbar war.

DNS = Domain Name System; verantwortlich für die Übersetzung von lesbaren Website Namen/Domains wie zB www.sunlime.at in eine numerische, maschinenlesbare Internet Adresse. Jedes mal wenn man ein E-Mail versendet oder eine Website aufruft, sendet der Computer einen DNS Look-up request an den Internet Service Provider, um den Datenverkehr an die richtige Adresse zu routen.

HTTPS

Jede neue Website sollte von Beginn an mit HTTPS gebaut werden.

— Scott Helme

Rüsten Sie Ihre Websites mit einem HTTPS Zertifikat auf.

HTTP

Hypertext Transfer Protocol

unverschlüsselter, ungeschützter Datenaustausch

HTTP (Hypertext Transfer Protocol) ist das Standardprotokoll für das Bereitstellen von Websites und für den Datenaustausch. Jedoch erfolgt dieser Datenaustausch nicht verschlüsselt und ungeschützt.

HTTPS

Hypertext Transfer Protocol Secure

Schützt Datenaustausch durch TLS

HTTPS kombiniert HTTP mit TLS (einem kryptografischen Protokoll; Transport Layer Security) und schützt Daten, die über das Netz ausgetauscht werden, durch Verschlüsselung.

Warum HTTPS?

Warum soll man HTTPS verwenden, wenn der Inhalt einer Website ohnehin öffentlich ist?

Warum HTTPS?

  • Verschlüsselt Passwörter
  • Schützt davor, falsche Inhalte zu erhalten
  • Beugt SQL Injections vor
  • Wenn man ein CMS wie zB Wordpress für seine Website verwendet, werden bei HTTP die Zugangsdaten in Reintext zum Server gesendet. Jeder Benutzer, der sich im entsprechenden WLAN befindet oder die Internetverbindung abhört (durch im Netz frei zugängliche Tools), kann diese Daten auslesen und sich dadurch leichten Zugang zum CMS verschaffen. Ganz gefährlich sind dabei öffentliche WLANs wie zB jene in Cafés, Restaurants oder Hotels.
  • Wenn HTTPS korrekt implementiert wurde, können Angreifer nicht mehr das DNS übernehmen, um andere Inhalte auf der Domain auszuliefern.
  • Illegal installierte Software auf Computern kann keine Werbung oder Malware auf Webseiten einspeisen.

Vorteile von HTTPS

  • Schneller als HTTP
Ein jeder Test lädt 360 unterschiedliche, nicht-gecachte Bilder mit einer Gesamtgröße von 0.62 MB.

Vorteile von HTTPS

  • Sicherer als HTTP
    • 1. Unterbindet Inhaltseinschleusungen von Dritten

1. Werbung wird über ein Öffentliches WLAN auf eine Website eingeschleust.

Vorteile von HTTPS

2. Böswillige Inhalte werden in den Browser des Besuchers eingeschleust.

  • Bessere SEO Rankings

Vorteile von HTTPS

  • Keine Browserwarnungen
Es ist möglich, dass Browser eine Warnung zeigen, wenn die Website über HTTP ausgeliefert wird.

Vorteile von HTTPS

  • Mehr Funktionen
Bestimmte Browser Features werden für HTTP gesperrt (zB Geräte-Orientierung, Fullscreen, Geolocation etc.)

Vorteile von HTTPS

  • Bessere Seitenverweisen

Passwörter

Wichtiger als die Länge ist uns, dass wir uns an unsere Passwörter erinnern können. Und genau das ist das Problem!
Wer verwendet 14 Zeichen starke Passwörter mit Sonderzeichen, Groß- und Kleinschreibung und Phrasen, die in keinem Wörterbuch stehen?

Die Komprimitierung von Passwörtern

  • Password lockouts: Trügerische Sicherheit

Wir fühlen uns relativ sicher, weil wir glauben, dass Hacker auf eine Website gehen, den Benutzernamen eingeben und versuchen, dass Passwort durch mehrfache Eingabe zu erraten.

Wir fühlen uns deshalb so sicher, weil wir uns selbst schon so oft aus unseren Konten ausgesperrt haben, weil wir das Passwort falsch eingegeben oder sogar vergessen hatten.

Die Komprimitierung von Passwörtern

  • Password lockouts: Eine trügerische Sicherheit
Wir glauben, dass es den Hackern gleich ergeht. Nach drei Versuchen werden sie, so wie wir, automatisch ausgesperrt - glauben wir - und wiegen uns in trügerischer Sicherheit.

Die Komprimitierung von Passwörtern

  • Password lockouts sind kein Schutz mehr
Lockouts nach 3 oder 5-maligen Fehlversuchen und ähnliche Schutzmaßnahmen stellen mittlerweile Hacker vor keine großen Probleme mehr. Sie wissen den Benutzername und das Passwort bereits bevor sie auf die Anmeldeseite der Website kommen. Denn sie haben das Passwort bereits Stunden, Tage oder Monate vorher offline gehackt.

Die Komprimitierung von Passwörtern

  • Offline Angriffe sind ein Paradies für Hacker
Online Hacks sind schwierig, aber offline Angriffe sind ein Paradies für Hacker. Das einzige Limit ist die Qualität von Software und Hardware, die ein Hacker verwendet. Geschickte Hacker können Milliarden von Versuchen pro Sekunde mit normalen Computern ausführen.

Die Komprimitierung von Passwörtern

  • Der typische offline Password Hack

über gestohlene Datenbanken

Der typische offline Passwort Hack erfolgt über eine Datenbank, die der Hacker über einen Einbruch auf einem Server gestohlen hat. Dabei zielt der Angreifer nicht auf individuelle Kontoinhaber ab, sondern auf die Menge und versucht einige wenige Passwörter zu entschlüsseln und darüber Zugang auf viele Konten zu erhalten.

Die Komprimitierung von Passwörtern

Hashfunktionen und Passwörter

echo "sunlime" | openssl sha1
949c913bef2ef05a8ce3d5cb9886d075ca7680d0

sunlime => 949c913bef2ef05a8ce3d5cb9886d075ca7680d0

Passwörter werden niemals als Reintext in Datenbanken gespeichert sondern werden über eine kryptographische Hashfunktion verschlüsselt.

Ein Hash ist ein Algorythmus, der ein Wort in eine nicht wieder erkennbare Zeichenkette mit einer bestimmten Länge verwandelt.

Die Idee ist, dass man eine gehashte Version des Passwortes bedenkenlos in einer Datenbank speichern kann. Außerdem gibt es keine kryptografische Möglichkeit, aus einer gehashten Zeichenkette das ursprüngliche Wort wiederherzustellen.

Die Komprimitierung von Passwörtern

Hashfunktionen und Passwörter

14 * 18 * 29 = 7.308

7.308 = ?

Außerdem gibt es keine kryptografische Möglichkeit, aus einer gehashten Zeichenkette das ursprüngliche Wort wiederherzustellen. Gleich wie es nicht möglich ist, aus dem Produkt einer Multiplikation, auf die Faktoren zu schließen.

sunline => 88fc0d6348ed734268a4a92e8e30698cb0854fd1

Eine weitere Eigenschaft eines Hashes ist, dass eine kleine Änderungen im Reintext eine große Änderung im gehashten String bedingt.
Dadurch wird es unmöglich, sich an ein Passwort durch schrittweises Ausprobieren von Zeichen anzunähern. D.h. man kann aus dem Hash von "sunlime" niemals auf den Hash von "sunline" schließen.

Die Komprimitierung von Passwörtern

Wie gehen Hacker vor?

Sie vergleichen Hash Werte in gestohlenen Datenbanken.

Die komprimitierte Datenbank enthält die Benutzernamen in Reintext und die zugehörigen Passwörter als Hash Wert. Die Hacker wenden nun den in der Datenbank für die Verschlüsselung von Passwörtern verwendeten Hash Algorythmus auf ein beliebiges Passwort an.

Dann vergleichen sie den Hash Wert dieses Passworts mit den in der Datenbank enthaltenen Hash Werten. Wenn sie in der Datenbank diesen Hash Wert finden, dann haben sie einen Treffer gelandet und der Benutzer hat ein Problem.

Sie greifen also nicht individuelle Benutzer an sondern versuchen von den Millionen von Benutzern in der Datenbank Passwörter zu entschlüsseln. Das heißt, sie zielen auf die niedrig hängenden Früchte ab.
