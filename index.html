HTTPS

Any new site or web application should be built with HTTPS out of the box.

— Scott Helme

Rüsten Sie Ihre Websites mit einem HTTPS Zertifikat auf. Hier eine

HTTP

Hypertext Transfer Protocol

Ungeschützter, unverschlüsselter Datenaustausch

HTTP (Hypertext Transfer Protocol) ist das Standardprotokoll für das Bereitstellen von Websites und für den Datenaustausch. Jedoch erfolgt dieser Datenaustausch ungeschützt und nicht verschlüsselt.

HTTPS

Hypertext Transfer Protocol Secure

Schützt Datenaustausch durch TLS

HTTPS kombiniert HTTP mit TLS (einem kryptografischen Protokoll) und schützt Daten, die über das Netz ausgetauscht werden, durch Verschlüsselung.

Warum HTTPS?

Warum soll man HTTPS verwenden, wenn der Inhalt einer Website ohnehin öffentlich ist?

Warum HTTPS?

  • Verschlüsselt Passwörter
  • Schützt davor, falsche Inhalte zu erhalten
  • Beugt SQL Injections vor
  • Wenn man ein CMS wie zB Wordpress für seine Website verwendet, werden bei HTTP die Zugangsdaten in Reintext zum Server gesendet. Jeder Benutzer, der sich im entsprechenden WLAN befindet oder die Internetverbindung abhört (durch im Netz frei zugängliche Tools), kann diese Daten auslesen und sich dadurch leichten Zugang zum CMS verschaffen. Ganz gefährlich sind dabei öffentliche WLANs wie zB jene in Cafés, Restaurants oder Hotels.
  • Wenn HTTPS korrekt implementiert wurde, können Angreifer nicht mehr das DNS übernehmen, um andere Inhalte auf der Domain auszuliefern.
  • Illegal installierte Software auf Computern kann keine Werbung oder Malware auf Webseiten einspeisen.

Vorteile von HTTPS

  • Schneller als HTTP
Ein jeder Test lädt 360 unterschiedliche, nicht-gecachte Bilder mit einer Gesamtgröße von 0.62 MB.

Vorteile von HTTPS

  • Sicherer als HTTP
    • 1. Unterbindet Inhaltseinschleusungen von Dritten

1. Werbung wird über ein Öffentliches WLAN auf eine Website eingeschleust.

Vorteile von HTTPS

  • Sicherer als HTTP
    • 2. Unterbindet Malware Einschleusung
2. Böswillige Inhalte werden in den Browser des Besuchers eingeschleust.

Vorteile von HTTPS

  • Bessere SEO Rankings
HTTPS ist seit 2014 ein Faktor in Google's Reihungsalgorhythmen

Vorteile von HTTPS

  • Keine Browserwarnungen
Es ist möglich, dass Browser eine Warnung zeigen, wenn die Website über HTTP ausgeliefert wird.

Vorteile von HTTPS

  • Mehr Funktionen
Bestimmte Browser Features werden für HTTP gesperrt (zB Geräte-Orientierung, Fullscreen, Geolocation etc.)

Vorteile von HTTPS

  • Bessere Seitenverweisen

Passwörter

Wichtiger als die Länge ist uns bei der Wahl unserer Passwörter, dass wir uns an sie erinnern können. Und genau das ist das Problem!
Während uns bewusst ist, dass Passwörter ein Sicherheitsrisiko darstellen, verwenden die Wenigsten von uns 14 Zeichen starke Passwörter mit Sonderzeichen, Groß- und Kleinschreibung und Phrasen, die in keinem Wörterbuch stehen.

Die Komprimitierung von Passwörtern

  • Trügerische Sicherheit
Wir fühlen uns relativ sicher, weil wir glauben, dass Hacker auf eine Website gehen, den Benutzernamen eingeben und versuchen, dass Passwort durch Mehrfache Eingabe zu erraten. Wir fühlen uns deshalb so sicher, weil wir uns selbst schon so oft selbst aus unseren Konten ausgesperrt haben, weil wir das Passwort falsch eingegeben oder sogar vergessen hatten.

Die Komprimitierung von Passwörtern

  • Password lockouts: Eine trügerische Sicherheit
Wir fühlen uns relativ sicher, weil wir glauben, dass Hacker auf eine Website gehen, den Benutzernamen eingeben und versuchen, dass Passwort durch Mehrfache Eingabe zu erraten. Wir fühlen uns deshalb so sicher, weil wir uns selbst schon so oft selbst aus unseren Konten ausgesperrt haben, weil wir das Passwort falsch eingegeben oder sogar vergessen hatten. Wir glauben, dass es den Hackern gleich ergeht. Nach drei Versuchen werden sie, so wie wir, automatisch ausgesperrt - glauben wir - und wiegen uns in trügerischer Sicherheit.

Die Komprimitierung von Passwörtern

  • Password lockouts sind kein Schutz mehr
Lockouts und ähnliche Schutzmaßnahmen stellen mittlerweile Hacker vor keine großen Probleme mehr. Sie wissen den Benutzername und das Passwort bereits bevor sie auf die Anmeldeseite der Website kommen. Denn sie haben das Passwort bereits Stunden, Tage oder Monate vorher offline gehackt.

Die Komprimitierung von Passwörtern

  • Offline Angriffe sind ein Paradies für Hacker
Online Hacks sind schwierig, aber offline Angriffe sind ein Paradies für Hacker. Das einzige Limit ist die Qualalität von Software und Hardware, die ein Hacker verwendet. Geschickte Hacker können Milliarden von Versuchen pro Sekunde mit normalen Computern ausführen.

Die Komprimitierung von Passwörtern

  • Der typische offline Password Hack
Der typische offline Passwort Hack erfolgt über eine Datenbank, die der Hacker über einen Einbruch auf einem Server gestohlen hat. Dabei zielt der Angreifer nicht auf individuelle Kontoinhaber ab, sondern auf die Menge und versucht einige wenige Passwörter zu entschlüsseln und darüber Zugang auf viele Konten zu erhalten.

Die Komprimitierung von Passwörtern

  • Das Problem mit Hashes
echo "sunlime" | openssl sha1

						949c913bef2ef05a8ce3d5cb9886d075ca7680d0
Passwörter werden niemals als Reintext in Datenbanken gespeichert sondern werden über einen Prozess namens Hash verschlüsselt. Ein Hash ist ein Algorythmus, der eine Wort in eine nicht wieder erkennbare Zeichenkette mit einer bestimmten Länge verwandelt. Die Idee ist, dass man eine gehashte Verson des Passwortes bedenkenlos in einer Datenbank speichern kann. Außerdem gibt es keine kryptografische Möglichkeit, aus einer gehashten Zeichenkette das ursprüngliche Wort wiederherzustellen.

Die Komprimitierung von Passwörtern

  • Das Problem mit Hashes
echo "sunlime" | openssl sha1

						949c913bef2ef05a8ce3d5cb9886d075ca7680d0
echo "sunline" | openssl sha1

						88fc0d6348ed734268a4a92e8e30698cb0854fd1
Eine weitere Eigenschaft eines Hashes ist, dass eine kleinere Änderungen im Reintext eine große Änderung im gehashte String bedingt.
Dadurch wird es unmöglich, sich an ein Passwort durch schrittweises Ausprobieren von Zeichen anzunähern. D.h. man kann aus dem Hash von "sunlime" niemals auf den Hash von "sunline" schließen.

Die Komprimitierung von Passwörtern

  • Wie gehen die Hacker vor?

Hash Werte vergleichen

Die komprimitierte Datenbank enthält die Benutzernamen in Reintext und die zugehörigen Passwörter als Hash Wert. Die Hacker wenden nun den in der Datenbank für die Verschlüsselung von Passwörtern verwendeten Hash Algorythmus auf ein beliebiges Passwort an. Dann vergleichen sie den Hash Wert dieses Passworts mit den in der Datenbank enthaltenen Hash Werten. Wenn sie in der Datenbank diesen Hash Wert finden, dann habne sie einen Treffer gelandet und der Benutzer hat ein Problem. Sie greifen also nicht individuelle Benutzer an sondern versuchen von den Millionen von Benutzern in der Datenbank Passwörter zu entschlüsseln. Das heißt, sie zielen auf die niedrig hängenden Früchte ab.
